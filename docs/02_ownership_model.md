# Ownership Model

## Ownership approach

The platform does not use a generic RACI model. Ownership is defined through:
- Capability Charters for modules
- Specialist Charters for walkers
- Workflow Charters for orchestrators
- a Decision Ledger for material cross-platform decisions

## Ownership roles

### Business Owner
Owns the business outcome.

### Technical Owner
Owns technical implementation and service health.

### Process Owner
Owns workflow shape and operating rhythm.

### Control Owner
Owns control sufficiency, challenge quality, and governance standards.

### Service Operator
Owns day-to-day operational running and incident triage.

### Consumer Lead
Represents primary users of the component.

## Rule

Every module and workflow must have one named owner for each relevant ownership dimension. Shared execution is allowed; diffuse accountability is not.
